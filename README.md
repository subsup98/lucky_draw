# 이치방쿠지 서비스 문서 세트

이 폴더는 이치방쿠지 스타일의 웹/앱 서비스를 기획하고 개발하기 위한 기본 마크다운 문서 세트입니다.

## 포함 문서
- `README.md`: 프로젝트 개요
- `requirements.md`: 기능/비기능 요구사항
- `security.md`: 보안 요구사항 및 운영 원칙
- `policy.md`: 이용정책, 환불, 배송, 금지행위
- `tasks.md`: MVP 기준 작업 분해
- `architecture.md`: 시스템 구조 초안
- `api.md`: API 초안

## 권장 사용 순서
1. `requirements.md`에서 범위 확정
2. `policy.md`, `security.md`로 운영/보안 기준 확정
3. `architecture.md`, `api.md`로 설계 구체화
4. `tasks.md`로 개발 일정 관리

## 현재 가정
- 실물 상품 배송형 온라인 쿠지 서비스
- 웹 우선, 앱 확장 가능 구조
- 사용자/운영자 기능 분리
- 서버 측 추첨 처리
